# 📚 Bookstore Management System | SQL Project

## 📖 Overview

This project demonstrates the design and implementation of a **Bookstore Management System** using **PostgreSQL**. It manages information about books, customers, and orders while showcasing SQL techniques for data retrieval, analysis, and reporting.

The project focuses on solving real-world business problems using SQL, including sales analysis, inventory tracking, customer insights, and revenue reporting.

---

## 🚀 Technologies Used

* PostgreSQL
* pgAdmin 4
* SQL
* CSV Dataset

---

## 📂 Database Tables

* **Books** – Stores book details such as title, author, genre, price, and stock.
* **Customers** – Stores customer information.
* **Orders** – Stores order history and connects customers with books.

---

## ✨ SQL Concepts Covered

* DDL (`CREATE`, `DROP`)
* Data Import (`COPY`)
* Filtering (`WHERE`, `BETWEEN`)
* Sorting (`ORDER BY`)
* Aggregation (`SUM`, `COUNT`, `AVG`)
* Joins (`INNER JOIN`, `LEFT JOIN`)
* Grouping (`GROUP BY`, `HAVING`)
* Date Functions
* Business Reporting Queries

---

## 📊 Business Insights

This project includes SQL queries to:

* Retrieve books by genre and publication year.
* Analyze customer orders and purchasing behavior.
* Calculate total revenue and inventory.
* Identify top-selling and most expensive books.
* Find high-value customers.
* Track remaining stock after sales.
* Generate sales reports by author and genre.
